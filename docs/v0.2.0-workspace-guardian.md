# SPM Dependency Tracker 0.2.0 Workspace Guardian

Version target: `0.2.0`
Tag target: `v0.2.0`
Main feature branch: `codex/feature/v0.2.0-workspace-guardian`

## Goal

Ship a monorepo-aware SPM dependency guardian that can:

- accept a repository root in addition to the existing single-target inputs
- discover auditable manifests and infer resolution contexts
- assemble an aggregate workspace report with partial-failure tolerance
- detect cross-manifest declared drift and cross-context resolved drift
- emit graph output with explicit provenance and bounded certainty
- render aggregate and graph-aware results in both the CLI and AppKit app

## Team Rules

- Keep public single-target behavior compatible unless the story explicitly changes it.
- Reuse existing single-target logic before introducing new abstractions.
- Prefer concrete structs and enums over protocol layers.
- Every new finding and graph artifact must carry enough provenance to explain why it exists.
- Partial failures must be recorded, never silently swallowed.
- Comments stay rare and explain only non-obvious policy or ownership logic.
- Tests are required for each story; no story is complete with code only.
