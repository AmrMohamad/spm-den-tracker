# Release 0.2.0

`0.2.0` is the first workspace-aware release of SPM Dependency Tracker.
The git tag for this release is `v0.2.0`.

## What Is New

- Repository-root analysis with explicit single-target and monorepo modes.
- Aggregate workspace reporting with per-context summaries and partial-failure visibility.
- Cross-manifest drift detection for declared requirements.
- Cross-context drift detection for resolved versions.
- Graph output with explicit provenance and bounded certainty labels.
- Documentation that explains the new workspace contract without changing the existing single-target baseline.

## What Stays Compatible

- The single-target audit flow remains available.
- Existing `doctor`, `report`, and `check-tracking` workflows remain the baseline user entry points.
- Release and installer scripts keep semantic versions in `x.y.z` form and release tags in `vX.Y.Z` form.

## Notes

- `Package.resolved` still provides resolved node metadata.
- Graph topology requires provenance from manifests, Xcode project data, or dependency-enrichment sources.
- Partial failures are surfaced in reports instead of aborting the entire workspace run where possible.
